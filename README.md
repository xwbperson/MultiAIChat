# AI Workspace

在一个 Windows 桌面窗口中集中使用多个 AI 网站，并为每个站点和账号隔离登录状态、代理与运行资源。

![AI Workspace 图标](build/icon.png)

## 功能亮点

- **多站点聚合：** 内置 ChatGPT、Claude、DeepSeek、Kimi、豆包、Copilot、Gemini 和 Perplexity，也可添加任意 HTTP/HTTPS 站点。
- **多账号隔离：** 每个账号使用独立的 Electron Session Partition，Cookie、缓存和登录状态互不影响。
- **Chrome 兼容环境：** 站点页面使用 Chromium 原生浏览器对象和统一的 Chrome User-Agent，不向网站暴露 Electron 或应用产品标记。
- **内存控制：** 通过 LRU 策略保留当前页和最近使用页，空闲页面延迟休眠并在再次访问时恢复原 URL。
- **灵活代理：** 支持直连、系统代理、HTTP/SOCKS5 默认代理，以及逐站覆盖；切换代理后会关闭旧连接并重载页面。
- **站点管理：** 支持添加、编辑、删除、排序、自定义图标、快捷键和账号名称。
- **稳定交互：** 添加站点采用单次提交并在后台缓存图标；提示、确认和文字输入均使用应用内对话框，关闭后恢复原控件焦点。
- **桌面体验：** 提供地址栏、前进后退、刷新、缩放、系统托盘、开机自启和全局呼出快捷键。
- **配置迁移：** 可导入或导出站点与设置；导入采用增量合并，并保留已有账号的会话隔离信息。
- **安全边界：** 渲染进程使用上下文隔离和受限 IPC，配置输入、URL、代理与本地图标均经过校验。

## 快速开始

```powershell
git clone https://github.com/xwbperson/MultiAIChat.git
cd MultiAIChat
npm install
npm start
```

执行完成后会打开 AI Workspace。首次使用各 AI 站点时，需要在对应页面自行登录；项目不要求环境变量或 API Key。

## 系统要求

- Windows 10 或 Windows 11（x64）
- Node.js 18 或更高版本（推荐当前 LTS）
- npm 9 或更高版本
- 能访问所选 AI 网站的网络环境；部分站点可能需要代理

## 使用方法

1. 从左侧栏选择站点；点击展开按钮可切换或添加独立账号。
2. 点击侧栏底部的站点管理按钮，添加自定义站点、调整顺序或设置站点快捷键。
3. 在设置面板选择默认代理；站点设置为“使用默认”时继承该值，也可单独指定直连、系统代理或代理地址。
4. 在性能设置中调整最大活跃页面数、空闲超时和休眠确认时间。休眠只销毁页面进程，不删除登录数据。
5. 使用设置面板导出配置进行备份；导出文件不包含 Cookie、缓存和账号登录凭据。

## 开发与构建

```powershell
npm run check
npm run build
npm run dev
```

`npm run check` 会执行 JavaScript 语法检查、Node.js 单元测试、隔离用户目录下的 Electron UI 回归测试和主进程安全退出检查。`npm run build` 会先生成应用图标，再在 `dist/` 中创建 Windows x64 NSIS 安装包。

主要代码位于以下目录：

- `src/main/`：窗口、WebContentsView、Session、代理、休眠、配置和 IPC。
- `src/renderer/`：侧栏、工具栏、设置、站点管理和界面样式。
- `test/`：配置仓库、会话代理、LRU、快捷键和休眠恢复单元测试。
- `scripts/renderer-ui-check.cjs`：设置滚动、活跃/休眠计数同步、站点编辑、连续拖拽排序和账号操作的 Electron UI 回归测试。
- `scripts/shutdown-check.cjs`：切换到正在加载的站点后立即退出，检查主进程是否发生未捕获异常。
- `docs/`：设计说明、功能完成情况和实现计划。

## 数据与隐私

应用配置保存在 Electron 的用户数据目录中。每个账号使用唯一的持久化分区；删除账号、删除站点或清除站点数据时，会同时清理相应会话和图标缓存。站点图标会保留远程来源地址，并在首次使用时下载到本地；后续界面只读取本地缓存。配置导出不会复制登录会话，但会保留图标来源地址，以便在另一台设备上重建缓存。

## 已知限制

- TUN 网卡自动检测、代理规则可视化和自动更新仍属于设计目标，当前版本尚未实现。
- 首次建立或重建默认站点图标缓存时需要访问 Google Favicon 服务；下载失败时显示 Emoji，并在后续启动时重试。
- 休眠状态不会跨应用重启保存，恢复页面仍可能受目标网站登录策略影响。
- 文档中的约 78% 内存节省是基于 15 个站点的理论估算，不是固定的实测结果。
