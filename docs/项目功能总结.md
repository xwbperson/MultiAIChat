> **版本：** 1.0.0  
> **最后更新：** 2026-07-13

---

## 一、项目概述

AI Workspace 是一款基于 Electron 的多 AI 站点聚合桌面客户端，将 ChatGPT、Claude、DeepSeek 等多个 AI 网站整合到一个窗口中，通过侧栏切换，提供统一、高效、隔离的多站操作体验。

## 三、技术架构

### 3.1 项目结构

```
MultiAIChat/
├── src/
│   ├── main/                    # 主进程
│   │   ├── main.js              # 应用入口
│   │   ├── preload.js           # 预加载脚本（IPC 桥接）
│   │   ├── window-manager.js    # 窗口状态管理
│   │   ├── view-manager.js      # WebContentsView 管理
│   │   ├── session-manager.js   # Session 分区管理
│   │   ├── config-store.js      # 配置存储
│   │   ├── default-sites.js     # 默认站点配置
│   │   ├── hibernation-manager.js # 休眠调度器
│   │   ├── tray-manager.js      # 系统托盘
│   │   ├── context-menu.js      # 右键菜单
│   │   ├── favicon-manager.js   # 图标管理
│   │   └── webview-preload.js   # WebView 预加载
│   │
│   └── renderer/                # 渲染进程
│       ├── index.html           # 主页面
│       ├── app.js               # 应用逻辑
│       ├── sidebar.js           # 侧栏组件
│       ├── toolbar.js           # 工具栏组件
│       ├── settings.js          # 设置面板
│       ├── site-manager.js      # 站点管理面板
│       └── styles/
│           └── main.css         # 全局样式
│
├── docs/                        # 文档
├── package.json
└── .gitignore
```

## 四、Session 隔离机制

### 4.1 Partition 命名规则

```
persist:<siteId>-<accountId>
```

示例：
- `persist:chatgpt-default` - ChatGPT 默认账号
- `persist:claude-personal` - Claude 个人账号

### 4.2 隔离内容

每个 Partition 完全独立：
- Cookie
- localStorage / sessionStorage
- IndexedDB
- Cache
- Service Worker 数据

## 五、标签页休眠机制

### 5.1 状态机

```
创建加载 → 活跃 → 空闲 → 待休眠 → 休眠
                ↑                    |
                └────────────────────┘  (点击唤醒)
```

### 5.2 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| maxActiveTabs | 3 | 最大同时活跃页面数 |
| idleTimeout | 30s | 空闲超时时间 |
| hibernateDelay | 10s | 休眠确认延迟 |

### 5.3 内存节省

- 无休眠：15 × 300MB ≈ 4.5 GB
- 有休眠：3 × 300MB + 12 × 5MB ≈ 960 MB（理论估算节省 78%，实际以站点和 Chromium 版本为准）

## 十、质量与安全保障

- `npm run check` 统一执行 JavaScript 语法检查、Node 行为测试和隔离用户目录下的 Electron UI 回归测试
- 自动测试覆盖 Session Partition 唯一性、配置导入合并、原子站点排序、LRU 休眠、休眠恢复、代理继承、WebContents 快捷键、设置滚动、连续拖拽排序、站点编辑及账号添加/重命名
- 站点和设置数据在主进程校验，账号 Partition 不允许通过通用站点更新接口覆盖
- 本地壳使用 CSP 与 HTML 转义防止导入配置污染渲染进程

---

## 十一、已知限制

1. 部分 AI 站点可能需要代理才能访问
2. 默认站点图标依赖 Google Favicon 服务；用户自定义图标会缓存到本地
3. 休眠状态在应用重启后重置
4. 暂不支持自定义主题
5. 设计文档中的 TUN 网卡自动检测、代理规则可视化和自动更新尚未实现
